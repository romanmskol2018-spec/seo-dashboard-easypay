# SEO Дашборд

Дашборд SEO-аналитики: трафик по сайтам (Яндекс.Метрика) и видимость по проектам (Топвизор).
Публичный дашборд + защищённая админка для ручного внесения данных.

## Технологии

- **Next.js 16** (App Router) + **React 19** + **TypeScript**
- **TailwindCSS 4** — стили
- **Recharts** — графики
- **Prisma 6** + **PostgreSQL** (Neon) — хранение данных
- **jose** + **bcryptjs** — авторизация (JWT в httpOnly cookie)

## Возможности

- 📊 Публичный дашборд: сводные метрики, графики трафика и видимости, таблицы по сайтам и проектам
- 🔀 Переключение периода: 7 / 30 / 90 дней (с динамикой к предыдущему периоду)
- 🔐 Админка (логин+пароль): управление сайтами, проектами и ручной ввод данных
- 🎨 Тёмная тема, адаптивная вёрстка

## Локальный запуск

1. Установи зависимости:
   ```bash
   npm install
   ```

2. Создай бесплатную базу на [neon.tech](https://neon.tech), скопируй строки подключения
   в `.env` (см. `.env.example`):
   - `DATABASE_URL` — пулерное подключение (с `-pooler` в хосте)
   - `DIRECT_URL` — прямое подключение (для миграций)

3. Создай таблицы и наполни демо-данными:
   ```bash
   npm run db:push     # создать схему в БД
   npm run db:seed     # демо: 6 сайтов, 6 проектов, 90 дней данных + админ
   ```

4. Запусти:
   ```bash
   npm run dev
   ```
   Дашборд: http://localhost:3000 · Админка: http://localhost:3000/admin

   Логин/пароль админа берутся из `.env` (`ADMIN_EMAIL` / `ADMIN_PASSWORD`).

## Полезные команды

| Команда | Действие |
|---|---|
| `npm run dev` | Запуск дев-сервера |
| `npm run build` | Production-сборка |
| `npm run db:push` | Применить схему Prisma к БД |
| `npm run db:seed` | Заполнить демо-данными |
| `npm run db:studio` | Prisma Studio (просмотр БД в браузере) |

## Деплой на Vercel

1. Залей проект в репозиторий GitHub.
2. На [vercel.com](https://vercel.com) → New Project → импортируй репозиторий.
3. В настройках проекта добавь переменные окружения (Environment Variables):
   `DATABASE_URL`, `DIRECT_URL`, `JWT_SECRET`, `ADMIN_EMAIL`, `ADMIN_PASSWORD`, `ADMIN_NAME`.
4. Deploy. После первого деплоя выполни миграцию и сидер для прод-базы
   (локально с прод-`DATABASE_URL` или через Vercel CLI):
   ```bash
   npm run db:push
   npm run db:seed
   ```
